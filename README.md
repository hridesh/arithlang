*Lang: A family of pedagogical interpreters
========

This repository contains the code for a chapter of the textbook *An Experiential
Introduction to Principles of Programming Languages* by Hridesh Rajan. Please
report errors and difficulties to the author at hridesh@acm.org.

This project builds with [Gradle](https://gradle.org) and targets **Java 21**.
The parser is generated with **ANTLR 4.13.2**.

# Organization

- `src/` — source code for the language (for example, `arithlang`). The ANTLR
  grammar (`*.g`) is included in the same directory as the hand-written sources;
  the parser it generates is placed in `src/<lang>/parser/` and is regenerated by
  the build.
- `lib/` — the bundled `antlr-4.13.2-complete.jar`.
- `build/` — generated class files and a copy of the example programs. This
  directory must be named `build` so the interpreter locates program files
  correctly at run time.
- `build.gradle`, `settings.gradle`, `gradlew` — the Gradle build.

# Requirements

A JDK 17 or newer must be installed (any recent JDK works; Gradle requires 17+).
On macOS, for example, `brew install openjdk@21`. Everything else is included:
the Gradle wrapper (`./gradlew` downloads Gradle automatically on first use) and
the bundled `lib/antlr-4.13.2-complete.jar`.

# Build and run

Generate the parser, compile, and start the interpreter (an interactive REPL):

    ./gradlew run

This opens the interpreter's `$` prompt: type a program and press Enter to see
its value (for example `(+ 1 2)`), or `run <file>` to load an example program.
Press Ctrl+C to exit. (The `$` prompt waits for your input; Gradle reports the
`:run` task as still "EXECUTING" the whole time it is open -- that is expected.)

Other useful tasks:

    ./gradlew generateParser   # (re)generate the ANTLR parser only
    ./gradlew build            # compile and assemble
    ./gradlew jar              # build a runnable fat jar under build/libs/
    ./gradlew clean            # remove the build/ directory

You can load an example program at the interpreter prompt; the example files are
copied to `build/<lang>/examples/` by the build.

# Using an IDE

The Eclipse project files (`.classpath`, `.project`, `.settings/`) are included.
Any modern IDE (Eclipse, IntelliJ IDEA, VS Code) can also import the project
directly as a Gradle project.
